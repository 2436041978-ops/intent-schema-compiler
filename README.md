# Intent Schema Compiler

> ⚠️ **这不是生产级框架。**
>
> 这是 [Schema-As-Code 思想体系](https://github.com/2436041978-ops/schema-as-code) 的一个**最小概念验证（MVP）**，
> 用来演示一件事：**设计意图可以被编译为机器可读契约，并在 LLM 输出进入界面前拦截语义漂移。**

## 这个 DEMO 展示了什么？

| 步骤 | 你看到的 | 背后的思想 |
|:--:|---------|-----------|
| 1 | 左侧 YAML 协议 | 自然语言规则（如"critical 不能用'严重'替代"）被形式化为机器可读契约 |
| 2 | 中间 JSON Schema | 协议被编译为结构化校验规则 |
| 3 | 右侧 LLM 输出 | 模拟 LLM 返回的生成内容 |
| 4 | 底部推演结果 | 四层校验（语法→语义→安全→美感）自动拦截漂移 |

**两种场景：**

- ✅ **合法输出**：LLM 严格遵守语义令牌，校验通过
- ❌ **违规输出**：LLM 用"严重"替代了"critical"，被同义词防火墙拦截

---

## 为什么做这件事？

在 AI 生成界面的时代，设计规范面临一个新问题：

> 以前规范写在文档里，开发者"应该"遵守；现在规范需要写在 LLM 的 Prompt 里，但 LLM 会"概率性"地不遵守。

这个 DEMO 证明了一种可能性：**把设计意图从"文档里的建议"变成"可编译、可校验、可拦截的契约"**，让语义漂移在到达用户之前就被阻断。

## 局限性与诚实声明

| 局限 | 说明 |
|------|------|
| **不是真实 LLM 接入** | 当前 DEMO 使用模拟数据演示校验逻辑，未接入真实大模型 API |
| **四层推演基于静态规则** | 语法、语义、安全层为规则引擎实现；美感层为简化评分 |
| **代码由 AI 辅助生成** | 作者为体验架构设计师，代码实现借助 AI 编程工具，仅供实验 |
| **不保证 API 稳定** | 本仓库为概念验证，接口可能随时调整，不建议作为依赖引入 |
| **仅覆盖告警卡片场景** | 当前协议定义围绕单一组件场景，未扩展至通用组件体系 |
