传统 Design Token

--color-danger: #D32F2F;
--spacing-md: 16px;
--radius-sm: 4px;
⚠️ 严重故障

仅定义视觉属性。LLM 不知道"严重"代表什么，也无法被机器校验。

→

Schema-As-Code 语义令牌

status.critical:
  canonical_id: "ST-001"
  immutable: true
  visual_mapping:
    color_token: "status.critical"
    motion_token: "pulse.red.urgent"
  llm_constraints:
    - "必须包含故障定位"
    - "禁止建议自动修复"
  synonym_firewall:
    prohibited: ["严重"]
P0
系统级故障
CPU 使用率超过阈值，导致服务响应延迟
置信度: 0.85 | 需人工确认

色值 + 语义 + LLM 约束 + 同义词防火墙。机器可读、可编译、可拦截。

三层协议结构

语义层 — 定义「应该有什么」

semantic_tokens:
  status.critical:
    description: "系统级故障，需立即响应"
    llm_constraints:
      - "生成内容必须包含明确的故障定位信息"

治理层 — 定义「不能突破什么」

human_ai_boundary:
  destructive-action:
    ai_prohibited:
      - "直接执行修复操作"
      - "修改告警阈值配置"

执行层 — 定义「怎么验证」

scenario_tests:
  - case: "同义词替代"
    mock_response: { alert_level: "严重" }
    expected_validation: "BLOCK"
